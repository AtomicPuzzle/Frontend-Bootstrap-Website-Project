3 Websites with Downloadable Resources

Boostrap (v3.3.7)

  • The boostrap library has many useful tools to help developers create websites with responsive, mobile-first designs. For this project I utilized Boostrap's navbar component and its grid system which helped me organize my elements and make easier style adjusments depending on the user's screen's width.
  • Boostrap also requires a jQuery to function. The reference link is required to be near the end of the <body> element after all the main content.

Font Awesome

  • FontAwesome is a font and CSS toolkit developed by Dave Gandy. It provides developers with free, open-source icons that can be used for commercial and personal projects. I use FontAwesome for my social media and contact links as well as leaf symbol next to food items that are vegan friendly.

Dynamic Drive

  • Dynamic Drive is a repository for DHTML and Javascripts that are free for anyone to use for their website (provided proper attribution is given). I downloaded a script that allows my webpage to scroll to a bookmark when the proper link is selected. This creates a smoother look as opposed to the page immediately jumping to a bookmark's location.
  • This script also requires a jQuery to function. The reference link is required to be in the <head> element.

w3schools.com

MDN Web Docs

Stack Overflow

HTML

Structural Elements

The navbar for each page is contained within a <header> element. The content for each page is separated into <section> elements. For example, on the INDEX page, the Lunch/Dinner menu, Brunch menu, and About Us are kept in their own section due them having different content and styles.

Anchor References & Links to Additional Information Pertaining to the Business

Anchor references and links are used throughout the project. For this example, I bring attention to the <header> and <footer> elements. The header contains links to different parts of the website (FOOD MENU, CONTACT page, DINNER SERIES page, MAIN page). The footer has links that open in new windows to the restaurant's social media sites, as well as a google maps location, and a link that opens up the CONTACT page as well.

Bookmarks

For this example, click on one of the links in the navbar at the top of the page. It is implemented with a script that scrolls to a bookmarked section on a current page or an external one.

Inline Images and Line Breaks

For inline images, I used one for the food menu on the index page and another to display the Special Dinner menu on the Dinner Series page. Line breaks are used in the About Us section to separate the paragraphs.

Apprpriate Marking for Elements

Marking was used to highlight certain information that I felt should catch the user's eye. There is a link to the Art Institute of Vancouver in the "About Us" section (this is where Chef Gordon went to culinary school). The upcoming Special Dinner menu has also been highlighted on the Dinner Series page.

Ordered, Unordered, and Description Lists

On the Dinner Series page, there is an ordered list of previous dinner menus. On the index page, in the first section, there is a small unordered list with the restaurants hours of operation. In the footer of each page, I used a description list for listing the restaurant's phone number, google maps link, and email contact link.

Special Character Sets

I used the "&, <, >" special character sets for this project. Featured in the food menu's heading and the "add on" feature for some items.

Implement HTML5 meta tags

Meta tags are located in the head element. If you are using the Chrome browser, press F12 or right click and select "Inspect." This will load up the Chrome Inspector which will allow you to view the HTML code and the appropriate meta tags in the head element.

Footer and Last Update

Every page has a <footer> at the bottom of each page. In the bottom-left corner, the "Last Update" date is shown (this was achieved using a Javascript technique on w3schools.com).

CSS

Inline Styles for <h1> Element

At the top of the index page, there is an <h1> element below the navbar that introduces the user to the website and the overall philosophy of the restaurant.

Embedded Stylesheet

An embedded stylesheet is located at the end of the <head> element of each page. This stylesheet is used to apply the 'Raleway' font (found on Google Fonts) to the entire page. If you are using the Chrome browser, press F12 or right click and select "Inspect." This will load up the Chrome Inspector which will allow you to view the HTML code and the appropriate meta tags in the head element.

External Stylesheets

In the source files, external stylesheets are located within the "Styles" folder. The primary external stylesheet is under the name "mainStyles.css"

Format Paragraph, List, Header, Background Images, Tables, and Forms

The stylesheets located in the "Styles" folder contain CSS code used to manipulate the above elements.

Styling with Colour

RGBA was used to style the frame surrounding the map in the <footer> as well as styling the shadow around the video in the "About Us" section. The border

between the main menu and brunch menu is coloured usig a progressive gradient.

Styling Text

Various styles including font-family, font-size, font-weight, font-style, text-decoration, and font-variant are used throughout each web page. For example, the main title in the main brand in top navbar.

Pseudo-Classes

The navbar in the <head> as well the form on the CONTACT page make use of visited, active, hover, and focus pseudo-classes. The <footer> makes use of the nth-of-type structural pseudo-class.

<div> and <span> tags

<div> elements are used throughout the entire project to aid with separating various sections of key information. This aids with styling the different sections in order to make them unique. The <span> element is a useful tool to style specific pieces of text. The bold "CHEF'S" word in the top navbar is encompassed by a <span> element.

Rotate Feature

The "NEW" and "Leaf" icons on the food menu make use of the CSS 'transform' feature using either 'matrix' or 'rotate'.

Drop Cap

The introductory message on the index.html page uses a drop cap styling for the "W".

Background & Border Styles

The footer uses styling for the background and borders of the social media, contact links and Google Maps elements (rounding corners, colouring background, resizing background, border size and padding).

Web Form

A form is found on the "Contact Us" page. It is created using a <form> element, within it are labels with corresponding inputs. CSS was used to position the inputs and labels in the proper place as well as making the inputs scale to the appropriate size when the broswer window changes size.

While the entire form uses a Javascript to validate each input box, the email and message boxes contain the 'required' attribute.

The focus pseudo-class is used to highlight the current boxed being selected. The focus and invalid pseudo-classses are also used on the email address and message boxes to test whether they contain valid inputs by making the border glow an additional shade of red.

Multimedia

Video

In the 'About Us' section on the INDEX page, there is a demo video for the restaurant. It is a <video> element with an additional <object> element nested within it (a contingency in case the video source does not load properly, in which case the <object> will take over).

Well Formed Document

The web pages utilize HTML5, CSS3, and Javascript.

Javascript

**The Javascript I wrote up is located inside the 'Javascript' folder and is within the 'main.js' file.

JS Features

The 'main.js' file makes use of variables, functions, array, comparison and logical operators, subroutine, conditional statements and operators, and time commands.

System Date

A Javascript is used to check the page and get the current Date object. The script then breaks the date up into smaller parts (day, month, year) and is placed inside an array which writes to the HTML page. The date is located at the end of the home page, index.html (and every other page), at the end of a footer element.

Validate Form Input (and Use of Alerts for Validation Purposes)

For the form element on the contact.html page, Javascript was used to test whether the fields had filled out with the proper inputs. Though some elements use the HTML 'required' attribute, the Javascript checks every part and will send an alert to the user (provided that the input has not yet received the 'required attribute').
